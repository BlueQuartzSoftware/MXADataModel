Basics

Download, compile and install CMake first. Then proceed to build all the third party libraries that are needed. See this page for more information regarding the necessary libraries.

Building

Use CMake to generate Makefiles for your platform. If you are running OS X and you want Xcode projects have CMake geneate those projects for you.
When linking against Boost use the static, multi-threaded versions of the boost libraries. There seems to be a bug where linking against the unit_test_framework dynamic library will cause either segmentation faults or the tests will not run properly

Boost 1.36.x Notes

The default compilation will yield dynamic libraries with no "install_name" set which means you will have to export the DYLD_LIBRARY_PATH with the appropriate folder value. If you hack the boost build system you can create static libraries which are easier to deal with. ./configure --with-libraries=thread,test,program_options,system,filesystem --prefix=/Users/Shared/Toolkits/boost-1_36
